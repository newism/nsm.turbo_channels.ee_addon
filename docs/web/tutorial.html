Specify custom fields

Uses the {custom_fields} parameter to include only the custom fields that are needed. In the following example we only want to include two custom fields to be included. The first two custom fields will be replaced as normal. However because the columns for the remaining custom fields are not included in the database query, the tags are not replaced as expected.

// input tag-data
{exp:channel:entries
  custom_fields="custom_field_1|custom_field_2"
}
  <h2>{title}</h2>
  <ul>
    <li>custom_field_1: <strong>{custom_field_1}</strong></li>
    <li>custom_field_2: <strong>{custom_field_2}</strong></li>
    <li>custom_field_3: <strong>{custom_field_3}</strong></li>
    <li>custom_field_4: <strong>{custom_field_4}</strong></li>
  </ul>
{/exp:channel:entries}

// output tag-data
// note that the third and fourth custom fields are not replaced.
// this is expected behaviour as we only requested the first and second fields in the custom_fields parameter
  <h2>Entry Title</h2>
  <ul>
    <li>custom_field_1: <strong>Value for custom field 1</strong></li>
    <li>custom_field_2: <strong>Value for custom field 2</strong></li>
    <li>custom_field_3: <strong>{custom_field_3}</strong></li>
    <li>custom_field_4: <strong>{custom_field_4}</strong></li>
  </ul>
{/exp:channel:entries}

NSM Turbo Channels is an extension that affects any tag or third party add-on that uses the Channel module and its related extension hooks.

The tag parameters listed on this page are the parameters that have been extended or are added by NSM Turbo Channels.

Parameters

disable [optional]

disable="channel_meta|view_counts"

Prevent columns of data from being retrieved in the database query. This parameter has been extended to include the following additional options.

Please be aware that by disabling chunks of data from the database query you are altering the behaviour of the Channel Entries. If your tags are not responding as you believe that they should, try adjusting the options in the disable parameter.

members

Including this option prevents the members table from being joined to the database query.

  • username
  • email
  • url
  • screen_name
  • location
  • occupation
  • interests
  • aol_im
  • yahoo_im
  • msn_im
  • icq
  • signature
  • sig_img_filename
  • sig_img_width
  • sig_img_height
  • avatar_filename
  • avatar_width
  • avatar_height
  • photo_filename
  • photo_width
  • photo_height
  • group_id
  • member_id
  • bday_d
  • bday_m
  • bday_y
  • bio
channel_meta

This option prevents the channels table from joining the database query.

  • channel_title
  • channel_name
  • channel_url
  • comment_url
  • comment_moderate
  • channel_html_formatting
  • channel_allow_img_urls
  • channel_auto_link_urls
  • comment_system_enabled
comments

If comments data is not required for the Channel Entries then include this option to omit the relevant table columns.

  • allow_comments
  • comment_expiration_date
  • recent_comment_date
  • comment_total
view_counts

Ignores database columns storing information related to view tracking.

  • view_count_one
  • view_count_two
  • view_count_three
  • view_count_four
entry_dates

Disables date-related database columns from the query.

  • dst_enabled
  • entry_date
  • year
  • month
  • day
  • edit_date
  • expiration_date
entry_meta

This option will not be used often as it disables database columns such as the title, url_title and status. It is most likely to be used in circumstances where only the entry_id is the only data that is needed.

  • forum_topic_id
  • author_id
  • ip_address
  • title
  • url_title
  • status
  • sticky
  • entry_site_id

db_ref [optional]

db_ref="eg Return all news items"

The value of the db_ref parameter is added as a comment to the SQL statement sent to the database. The value is processed by a regular expression to only send a safe string to the database. Expect characters to be removed from the string value.

custom_fields [optional]

custom_fields="my_custom_field|another_custom_field"

Specify a pipe-delimited list of custom fields that will be included in the database query. Choosing only the data columns that you need will yield a performance improvement by trimming un-needed data.

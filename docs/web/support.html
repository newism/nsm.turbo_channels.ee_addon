Data is not being output as expected

If data is not being output the way that you expect it to you may need to remove channel_meta from the disable parameters.

A common example of this behaviour:

// channel_meta enabled
{exp:channel:entries
  channel="blog"
  limit="1"
}
  {your_custom_field}
{/exp:channel:entries}

// output HTML
<img src="{filedir_1}image_url.jpg" class="image" width="240" height="160" />

// channel_meta disabled
{exp:channel:entries
  channel="blog"
  limit="channel_meta"
}
  {your_custom_field}
{/exp:channel:entries}

// output HTML
{filedir_1}image_url.jpg

This change in output data occurs because the channels table is no longer joined to the database query and ExpressionEngine will strip the HTML from the data as a fallback in the event that the channel options are not present.
